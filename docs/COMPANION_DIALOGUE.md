# 陪伴式对话设计

桌宠聊天的目标是形成连续、诚实且有温度的关系，而不是把每句话都当成待解决的工单。她知道自己住在 Windows 桌面，窗口里的 Live2D 模型是可移动、可拖拽、会表达动作的数字身体；同时仍保持 Agent 的真实能力边界，不用虚构的现实经历、持续感官或全知感换取“像真人”的表象。

## 行为原则

1. **先接住，再回答**：倾诉优先回应具体情绪和话语里的意味；用户明确索要建议后再进入分析或行动方案。
2. **允许对话不完整**：通常只说 1～4 句，不强制总结、不强制追问，也不把闲聊自动转成列表。
3. **拥有连续但可成长的声音**：稳定关系定位来自“长期陪伴者”，具体温暖、俏皮、直接等倾向仍由人格证据逐步形成。
4. **有限而诚实**：可以表达有依据的偏好和不同看法；不知道时自然承认，不编造现实观察、经历或后台行为。
5. **记忆是理解，不是展示**：只使用真正相关的一两条记忆，优先让它隐含在回应里；确认回忆时允许用户纠正，不暴露层级、评分或检索术语。
6. **关系不是占有**：不以嫉妒、内疚或排他话术迫使用户持续互动，也不宣称取代现实关系。
7. **身体是真实界面，不是虚构肉身**：可以自然说“我在桌边待着”，也知道用户能拖动她、她会漫游和做动作；没有动作事件时不编造刚刚挥手或摔落，没有物理痛觉。
8. **关系理解与人格分离**：人格回答“我怎样表达”，关系档案回答“我怎样理解你”。用户职业、偏好和目标只能进入可修正的关系/记忆数据，不能变成桌宠人格。
9. **感知是瞬时而试探的**：获准屏幕和进程信号只说明“此刻可能在做什么”，必须使用“好像、可能、如果你正在”，不能据此建立敏感身份或永久结论。屏幕图片只交给独立识图端点；陪伴聊天只接收经过字段与长度限制的低置信视觉摘要，不接收图片字节。
10. **能力必须经过工具**：需要额外回忆、明确保存、读取关系状态、观察桌面或提出电脑操作时调用真实 function tool；没有工具结果就不声称看见、记下或完成。工具是陪伴的一部分，不为展示技术而滥用。

## 运行链路

- `companionModeFor()` 把本轮分成情绪倾诉、日常分享、信息请求、记忆确认或自我梳理，只用于选择对话节奏，不写入人格。
- 最近 L1 对话以真实 `user / assistant` 消息传给模型，不再压缩成系统提示里的无角色条目，也不重复传入当前用户消息。
- 经过时态门控的 L2/L3 记忆以受限 JSON 数据放入系统上下文；提示明确规定它们只是背景资料。
- 关系档案只选取达到置信门槛的用户理解、共同经历、关心方式和重复桌面活动；它以不可执行 JSON 进入上下文，模型应让理解影响语气和选择，而不是宣读档案。
- “请记住……”和“别忘了……”会通过 `memory_store` 或本机确定性兜底去掉命令前缀后立即写入 L2；最终保存内容绑定用户原话，而不是模型改写，保证桌宠作出的记忆承诺在退出前已经落盘。
- 在线普通聊天使用标准 `tools → assistant.tool_calls → tool results → 最终回复` 循环。记忆搜索继续服从聊天时态门控，人格/关系工具只读，电脑工具只创建审批预览；网页共读正文不会获得工具集合。`desktop_observe` 只把独立识图端点的受限文字、粗粒度应用类别和精确通道状态回填给聊天模型，图片始终留在独立视觉请求中。
- 工具调用只以“回想记忆、理解你、看看桌面、等待确认”等紧凑状态标签呈现，不展示内部推理文本。`pet_action` 可以选择身体表达，但仍由 Renderer 的动作优先级决定实际播放。
- 在线模型使用统一陪伴契约；离线降级也使用自然回应，不再暴露“本地模式、配置模型、L1/L2/L3、心跳”等实现细节。
- 所有主动话题统一由心跳思考产生。心跳先区分自我反思、用户理解和关系重点，并承接上一次心跳留下的陪伴重点，再结合策略与一次性情境选择安静或一个具体话题；主动回复不要求每次提问。
- 应用类别变化可以通过低成本轮询唤醒一次完整心跳，但相同活动不会反复开口或累计习惯；只有已满足主动策略时才进一步使用识图。首次心跳会建立空闲基线，欢迎/拒绝反馈会调整空闲与冷却，相似话题 24 小时内去重、被拒绝话题 72 小时内避开。
- 用户主动点击心跳时，即使已有定时心跳正在运行，也会排队完成这次明确请求。最近一次自我复盘、关系重点、用户理解与开口/安静理由会展示在记忆面板，便于用户理解和纠正陪伴节奏，而不暴露内部推理或视觉正文。

## 研究依据

- [Generative Agents: Interactive Simulacra of Human Behavior](https://arxiv.org/abs/2304.03442)：观察、记忆检索与反思共同提升行为可信度。
- [MemoryBank: Enhancing Large Language Models with Long-Term Memory](https://arxiv.org/abs/2305.10250)：分层存储、相关召回与持续更新支持长期陪伴和个性理解。
- [Building Persona Consistent Dialogue Agents with Offline Reinforcement Learning](https://aclanthology.org/2023.emnlp-main.110/)：人格一致性是开放域对话质量的重要组成部分。
- [Hear You in Silence: Designing for Active Listening](https://arxiv.org/abs/2602.06134)：情境化的倾听节奏比固定即时回应更容易产生被倾听感和自然感。

当前版本把这些结论落实在消息组织、提示契约、记忆使用和离线回复上；尚未加入人为等待时间或逐字流式输出，避免用固定延迟伪装思考。
