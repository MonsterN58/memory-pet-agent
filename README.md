# 记忆桌宠 Agent

一个面向 Windows 的本地优先桌宠 Agent。当前 MVP 已具备透明置顶窗口、三级记忆、人格与关系成长、统一心跳思考、授权式桌面感知、受约束的主动关心、语音输入/输出、OpenAI 兼容模型与标准 function tool 调用，以及可替换的 2D 模型适配层。

## 已实现能力

- L1 瞬时记忆：保留当前会话最近消息，只存在于进程内。
- L2 海马体待整理区：心跳按容量或停留时间把 L1 对话组成事件并持久化。
- L3 长期记忆：从 L2 提炼事件、事实、偏好和反思；相似内容自动合并。
- 记忆检索：综合文本相关度、重要度、时间衰减和访问频率排序；中文检索会削弱高频虚词干扰，并在缺少最低文本证据时返回空结果。聊天还会按“现在 / 以前 / 变化”等显式时态选择当前、历史或比较视图，避免明确旧值进入当前回答或被访问次数反复强化。
- 记忆可控性：控制面板可展开 L1/L2/L3 完整内容、标签、时间与来源链；L2/L3 可修正类型、内容和重要度，也可确认后删除。搜索结果会拆分显示“为何召回”的各项分值。
- 心跳机制：统一执行短时情境采集、L1→L2、人格复盘、关系理解、L2→L3 和私有思考，再决定本轮开口或安静；心跳会承接上一次关系重点，控制面板还会展示最近一次自我复盘、用户理解、关系重点与开口/安静理由。
- 主动关心：所有主动话题只从心跳管线产生，同时受空闲阈值、冷却时间、每日上限、安静时段和“是否值得打扰”判断控制。首次启动也会建立空闲计时基线；欢迎/拒绝反馈会调整之后的阈值，相似话题会在 24 小时内去重，被拒绝的话题会避开 72 小时。
- 人格成长：持久化人格初始为空，通过重复对话证据和心跳复盘逐渐形成温暖、好奇、俏皮、直接、主动、表达六个连续倾向；模型身体只提供不入库的低置信起点，支持逐维证据覆盖、相反反馈修正、置信度、证据门槛和一键重置。
- 关系成长：独立的 `relationship-profile.json` 逐渐整理用户亲口表达的身份、偏好、目标、习惯、兴趣、困扰、工作方式、关心偏好和共同经历；它与桌宠自身人格分开，支持冲突修正、来源、置信度、主动话题反馈、设置页查看和单独重置。
- 陪伴式对话：明确知道自己是住在 Windows 桌面、拥有 Live2D 数字身体的桌宠，而不是客服或全知工具；会区分倾诉、闲聊、信息请求和回忆确认，先回应情绪与意味。用户可见正文只保留真正说出口的话；`（笑）`、`（歪头）`、`[emotion: happy]` 等舞台提示会在写入 L1 和情绪驱动前清洗，笑意改成自然的“哈哈哈”，表情与动作继续走独立字段。记忆优先体现为少问已知问题、接住进展和调整关心方式，不强制复述旧事。
- 桌面感知：屏幕理解与可见应用检测分别授权、默认关闭。屏幕模式只在普通聊天明确需要并调用 `desktop_observe`、用户手动心跳，或定时心跳已具备主动靠近资格时截取一张压缩缩略图并发送到独立识图端点；聊天 LLM 只收到识图端点清洗后的受限文字，绝不接收图片字节。进程模式使用快速本机只读查询映射已知粗粒度活动，窗口标题、PID 和原始输出不发送也不保存。
- 语音交互：默认使用项目内 sherpa-onnx + 小型中文 Zipformer 模型离线识别，启动后非阻塞预热；设置页会分别显示模型文件与运行时状态，预热失败后点击麦克风即可重试。识别阶段可立即取消，超时或 worker 故障会自动收尾并重建。朗读默认使用本机已安装语音，可选独立云端 TTS。
- 完整 Agent 工具循环：普通聊天向兼容模型提供标准 `tools / tool_calls / tool` 协议；模型可以按需搜索或明确保存记忆、读取人格/关系状态、进行一次性桌面感知、生成电脑操作预览和选择桌宠动作。主进程校验每份参数并把结果回填模型，最多循环四轮后生成最终回复。
- 模型接入：聊天、屏幕识图和可选云端 TTS 使用三组完全独立的 OpenAI 兼容配置。聊天端点支持 Chat Completions 与 function tools；识图端点单独接收标准 `image_url`，只返回当前任务、忙碌程度和帮助机会等受限 JSON。任一端点未配置或失败时，其余文字陪伴与本机能力继续工作。
- 桌宠窗口：默认只显示透明背景上的宠物本体；失去焦点后在当前桌面工作区自主行走，托盘常驻。
- 拖拽与落地：按住宠物移动即可把它拖到当前显示器任意位置；模型会按拖动方向、速度和上提幅度产生连续滞后与拉伸。松手后窗口垂直加速下落，模型姿态保留松手瞬间的水平速度倾斜，并播放约 320ms 的压缩回弹。
- 低遮挡对话贴片：鼠标靠近只显示位于模型上方的双行字幕，点击宠物、字幕或托盘唤醒后才显示桌宠最新回复和单行紧凑输入。字幕、回复、输入和活动状态已整体放大并提高对比度，保留原 226/254px 宽度与高度边界，不增加对模型的遮挡。支持多行自动增高、`Enter` 发送、`Shift + Enter` 换行、录音/识别/思考反馈以及全局 `Esc` 收起。
- 可见工具状态：本轮记忆、人格、关系、感知和动作调用以最多四个紧凑标签呈现；电脑操作进入独立确认态时隐藏重复历史、状态条和输入，只保留固定参数预览与授权按钮，确认或拒绝后恢复输入。
- 右键控制：自由移动、置顶、语音、心跳、桌面感知、记忆与高级设置全部从宠物右键菜单进入。
- 电脑协作：默认关闭；开启后可通过网页右键扩展、剪贴板快捷键或原生文件选择器，把用户明确选择的内容交给桌宠解释、总结、记住或继续聊天。网页内容始终按不可信数据隔离。
- 受控工作工具：模型十五项 function tools 与离线确定性规划器都能生成网页、剪贴板、文本文件、白名单应用、当前浏览器标签页和 Office 写入的固定参数预览；浏览器可刷新、前进/后退、上下滚动、滚到顶部/底部和页内查找，Office 可向当前 Word 追加、从 Excel 指定单元格写入纯文本表格、向 PowerPoint 新增文本页。明确的复合请求可形成 2～4 步顺序计划，全部参数先校验，再一次展示一步并逐步确认；任一步拒绝、取消或失败都会停止后续。真正参数只保存在主进程，执行结果回到 L1 供下一轮承接。
- Live2D 模型：内置 Hiyori、Mao、Wanko、Haru、Mark、Rice 六套 Live2D 官方样例，以及公开授权的 Nana、Cyannyan、小云三套第三方模型；可在右键菜单或设置面板中原子热切换，加载失败时保留当前模型，连续选择以最后一次为准。
- 自定义模型：可导入 Cubism 3/4/5 的 `.model3.json + .moc3` 整个文件夹，自动收集贴图、motion、expression、physics、pose、userdata 和动作音频。
- 待机与视线：九套模型待机时都有轻微呼吸/浮动；正式 Electron 页面只使用主进程桌面全局鼠标位置，按水平 640px、垂直 480px 半径渐进驱动标准眼球/头身参数，也兼容 Wanko 一类旧式头身和耳朵参数。
- 情绪与动作：回复会本地推断开心、兴奋、思考、好奇、安慰、害羞、惊讶、困倦等状态，并克制地选择至多一个语义动作。等待模型超过 650ms 时会平滑进入视线左上、轻歪头和低幅呼吸组成的通用思考姿态；回复、录音、拖拽、下落、落地或手动动作会及时撤下。右键菜单可分组预览 18 项动作，新增鞠躬、鼓掌、探看、沉思和展示结果等协作表达；手动预览优先于自动动作。
- 动作适配：Hiyori/Mao/Wanko 使用完整逐模型 motion 映射，Haru/Mark/Rice 只绑定语义合理的真实动作，其余动作和无 motion 的 Nana/Cyannyan/小云走平滑程序化反馈；Haru、Mao、Cyannyan、小云还有独立情绪 expression 映射。口型可从真实标准参数安全兜底，无嘴部参数时改用轻微身体脉冲。
- 模型初始气质：九个身体分别提供不同的六维低置信表达起点。它只在人格证据不足的维度生效，不写入人格档案；真实互动证据会逐维覆盖，切换模型不会清空已成长人格、关系或记忆。
- 2D 适配：通过 `PetModelAdapter` 隔离模型 Runtime 与 Agent、记忆、心跳和桌面移动。

## 本地运行

要求 Node.js 22 或更高版本。

Windows 下最简单的方式是双击项目根目录的 [`启动桌宠.cmd`](./启动桌宠.cmd)。脚本会自动定位项目目录、检查 Node.js、在缺少依赖时执行安装，并完成构建和启动。关闭或右键隐藏桌宠时会立即停止麦克风与朗读，并隐藏到系统托盘；从托盘菜单选择“退出”才会完全结束进程。

也可以在 PowerShell 中手动运行：

```powershell
npm install
npm run voice:model:download
npm start
```

离线识别 Runtime 随 npm 依赖安装，约 21 MB；中文模型约 29.5 MiB，固定保存到
`<项目目录>\resources\voice\sherpa-onnx-streaming-zipformer-zh-14M-2023-02-23`。
项目的 `.npmrc` 会把 npm 缓存固定到项目内 `.cache/npm`，因此模型和缓存都跟随项目所在磁盘；当前维护工作区在 D 盘时不会使用 C 盘用户 npm 缓存。
当前模型已下载时脚本只做 SHA-256 校验，不会重复下载；可用
`npm run voice:model:verify` 手动检查完整性。双击启动脚本发现模型缺失时会自动下载到该项目目录。

如果在国内网络下载 Electron 超时，可以使用镜像：

```powershell
$env:ELECTRON_MIRROR='https://npmmirror.com/mirrors/electron/'
npm install
```

开发检查：

```powershell
npm run typecheck
npm test
npm run test:memory-quality
npm run build
npm run smoke
npm run smoke:voice
npm run smoke:voice-ui
npm run smoke:model-switch
```

`npm run smoke` 会启动真实 Electron 运行时，验证 preload、IPC、托盘和本地存储初始化，随后自动退出且不显示窗口。`npm run smoke:voice` 会在真实 Electron 主进程中加载项目内 ONNX 模型并识别官方中文测试音频，不访问麦克风或网络。`npm run smoke:voice-ui` 使用项目 WAV 作为 Chromium 虚拟麦克风，覆盖 Renderer 录音、IPC、worker 成功识别、识别中立即取消、迟到结果隔离和下一次恢复。`npm run smoke:model-switch` 会显示透明桌宠窗口，依次热切换全部九套内置模型并再次回到 Hiyori，全部就绪后自动退出。

当前最近一次完整 `npm test` 为 208/208 通过，包含九模型注册表、MOC3 v1～v5 边界与气质种子、十五项 Agent tool 契约、18 项动作、用户可见正文清洗/L1/情绪驱动契约、浏览器 8 个固定动作与断线重投、Office 参数/事件边界，以及 2～4 步计划的原子创建、主进程顺序与同 ID 幂等、逐步审批、异步队列竞态隔离和停止后续语义。

`npm run test:memory-quality` 会运行 20 个手写中文记忆 fixture，覆盖偏好更新、事实冲突、跨天跟进、持久化提示注入和用户纠错，并验证聊天的当前、历史、比较三种上下文视图。回复级契约使用真实 `AgentService`、临时本地 JSON 和仅监听 `127.0.0.1` 的受控 Chat Completions 服务，验证旧偏好不会进入提示词、记忆引用或访问强化；不调用外部模型或网络。常规 `npm test` 也包含这组质量契约。

构建 Windows 安装包：

```powershell
npm run package
```

当前本地打包直接复用 `node_modules/electron/dist`，不会再次下载 Electron；生成物位于 `release/`。MVP 安装包明确关闭了 EXE 资源编辑与代码签名，便于无证书环境稳定构建。正式发布时应配置应用图标和 Windows 代码签名，并恢复 `build.win.signAndEditExecutable`。第三方 Runtime 声明随包附带在 [`THIRD_PARTY_NOTICES.md`](./THIRD_PARTY_NOTICES.md)。

## 电脑协作与网页右键

电脑协作默认关闭，需要在“桌宠设置 → 电脑协作、网页共读与权限”中主动开启。最轻量的入口是先在任意应用复制一段文字，再按 `Ctrl+Shift+E`，或从宠物/托盘右键菜单选择“解释、总结、聊聊剪贴板文本”。“让她阅读文本文件”只会读取原生选择器中由用户点选的 `.txt/.md/.json/.csv/.tsv/.log/.html/.xml` 文件，大小上限为 512KB，单次送入对话的正文最多 12000 字。

Chrome / Edge 网页右键接入步骤：

1. 同时开启“电脑协作”和“浏览器右键共读桥接”。桥接只监听 `127.0.0.1:32145`。
2. 点击“打开扩展目录”；在 Chrome 的 `chrome://extensions` 或 Edge 的 `edge://extensions` 开启开发者模式，选择“加载已解压的扩展程序”。
3. 点击设置里的“复制配对信息”，打开浏览器工具栏中的“记忆桌宠 · 网页共读”扩展，粘贴并测试。若要让操作卡控制当前页，再主动勾选“接收桌宠中已审批的网页操作”。
4. 在网页或选中文本上右键，即可选择“解释、总结、记住、聊聊”。扩展只在这次明确点击后读取对应选区；“总结当前网页”会裁剪正文到 12000 字，并排除表单、输入框、脚本、导航与页脚。当前页控制支持刷新、前进/后退、向上/下滚动、滚到顶部/底部和最多 200 字的页内查找；浏览器撤销 `activeTab` 临时权限后，需要再点击一次扩展图标。

配对使用随机的主机本地令牌；重新生成后旧令牌立即失效。浏览器桥没有记忆、设置、文件或页面数据读取接口；命令接口只能投递桌宠中已确认的一次性固定动作，并在 45 秒后过期。打开网页、复制和启动应用可配置询问/长期允许/禁止；保存文件、当前页控制与 Office 写入只提供本次确认或拒绝。保存文件随后还要在系统保存窗口中选择最终路径。所有提交和工具结果保存在 `data/computer-access.json` 的最近 500 条本地审计中，可在设置页查看和清空。

Office 桌面交互要求 Windows 上已安装并打开对应 Microsoft Office 应用和目标文件。可以直接说“在 Word 中追加：会议结论”“在 Excel B2 写入：姓名<Tab>状态…”或“在 PPT 添加一页：标题：周报；正文：本周完成三项任务”。确认后固定 COM 自动化只编辑当前活动文档，不主动调用宏、不打开任意文件，也不自动保存；Excel 写入期间会临时关闭并恢复事件，Word/PowerPoint 已有文档事件仍可能由 Office 自身响应。请在 Office 中检查结果并自行保存。更详细的边界见 [`docs/COMPUTER_INTERACTION.md`](./docs/COMPUTER_INTERACTION.md)。

需要连续处理时，可以明确说“打开 https://example.com，然后向下滚动网页，再在当前网页查找：安装”。支持 tools 的聊天模型可调用 `computer_work_plan`，本地兼容模式也会按“然后 / 接着 / 随后 / 再 / 分号”拆分全部可识别的 2～4 个步骤。桌宠会显示“步骤 1/3”这类确认卡，当前步骤完成后才展示下一步；拒绝任一步、取消系统保存窗口、操作失败或预览过期都会停止余下步骤。已经完成的网页、剪贴板、文件或 Office 操作保持原结果，不提供自动撤回。

## 模型配置

桌宠默认以本地陪伴模式运行，不要求 API Key。聊天 LLM、屏幕识图和可选云端 TTS 在设置页中分成三组，互不依赖。

聊天 LLM 组填写聊天 Base URL、聊天模型和聊天 API Key，并开启“启用模型服务”。也可以使用 `OPENAI_API_KEY`、`OPENAI_BASE_URL` 和 `OPENAI_MODEL` 环境变量，环境变量优先。完整 Agent 行为要求端点兼容 Chat Completions 的 `tools`、assistant `tool_calls` 与 `tool_call_id`；若端点只支持普通文本消息，桌宠会自动使用兼容聊天模式，明确记忆、明确桌面观察请求和常见电脑工作句式仍由本机规则兜底。

屏幕识图组单独填写识图 Base URL、视觉模型和识图 API Key，并开启“启用独立识图服务”。视觉模型需要兼容 `POST {识图 Base URL}/chat/completions` 和 user 消息中的 `image_url` 数据 URL；也可以使用 `OPENAI_VISION_API_KEY`、`OPENAI_VISION_BASE_URL` 和 `OPENAI_VISION_MODEL` 环境变量。该端点只接收当次压缩缩略图，返回经过字段和长度限制的当前场景、当前任务、忙碌程度、帮助机会与置信度；聊天 LLM 永远不会收到图片。识图配置缺失、超时或返回错误时，桌宠会明确说明是哪条感知通道未完成，并继续使用本地记忆、关系和应用类别思考。

朗读默认选择“本机语音”，不需要 API Key，也不会把回复文本发送到外部服务。需要更自然的声音时，可切换到“云端 TTS”，单独填写 TTS Base URL、模型、音色、语速和 TTS API Key；云端默认请求 `POST {TTS Base URL}/audio/speech`，也可以使用 `OPENAI_TTS_API_KEY`、`OPENAI_TTS_BASE_URL`、`OPENAI_TTS_MODEL` 和 `OPENAI_TTS_VOICE` 环境变量。云端配置缺失、请求失败或音频无法播放时会自动使用本机语音继续朗读。

聊天、屏幕识图和可选云端 TTS 的三份 API Key 都使用 Electron `safeStorage` 绑定当前系统用户加密保存，Renderer 只能看到各自是否已配置。旧版本升级时，已有 TTS 迁移规则继续生效；若旧设置曾在聊天模型启用时开启屏幕理解，聊天端点、模型和加密 Key 会一次性迁移到识图组，之后三份配置可以分别修改或清除，互不覆盖。

## 人格成长

桌宠不再使用可手填的“性格设定”字符串。首次启动时 `personality-profile.json` 中没有任何特质，第一轮回复采用中性表达；之后系统按以下过程成长：

1. 每次用户对话只提取交流方式信号，例如希望更直接、更主动、不要卖萌或需要详细解释，不把用户身份和普通事实当成桌宠人格。
2. 特质以连续分数保存，同时记录证据次数、置信度和最近证据。单次对话只能造成小幅变化，达到设置中的证据门槛后才参与回复。
3. 心跳对尚未处理的新 L2 对话做一次去重复盘。有模型时要求模型只返回受限 JSON 证据，无模型时使用本地规则。
4. 相反反馈会降低原方向置信度并推动分数回摆，不会永久锁定第一次判断。
5. 当前成长阶段、六项倾向和证据可以在“桌宠设置 → 身份与人格成长”查看，也可以单独清空人格而保留三级记忆。

最终仍需把结构化人格状态作为本轮模型上下文的一部分，这是语言模型执行行为的必要接口；区别在于该上下文来自持久化状态、累计证据和置信度计算，而不是一段预先写死的角色提示词。

## 关系档案、心跳思考与桌面感知

人格回答“她逐渐成为怎样的桌宠”，关系档案回答“她逐渐如何理解你们之间的相处”。普通聊天会同时收到两份结构化背景：关系资料只在置信度足够时参与回复，并被明确标记为可修正数据，因此桌宠会用它少问重复问题、自然跟进变化和选择更合适的关心方式，而不是把用户档案逐条念出来。下一轮用户对主动话题表现出的欢迎、普通回应或拒绝，也会进入本机关系状态，调整之后的主动程度。

心跳是主动行为的唯一决策入口，顺序固定为：

```text
采集获准的一次性桌面情境
  → L1 迁移到 L2
  → 复盘“我正在形成怎样的表达方式”
  → 整理“我对使用者和关系了解了什么”
  → L2 提炼到 L3
  → 形成私有心跳思考
  → 应用安静时段 / 空闲 / 冷却 / 日限额
  → 选择安静，或只生成一个自然话题
```

除完整定时心跳外，开启应用检测后还会按设置的间隔（默认 2 分钟）执行低成本活动轮询。轮询只在出现新的粗粒度活动类别时唤醒完整心跳；相同类别持续存在不会反复开口或强化习惯。只有当前已经满足主动开口策略时，活动变化唤醒的心跳才会额外请求一次识图，避免周期性截图和 API 消耗。若完整心跳仍在运行，用户点击“立即心跳整理”会排队补跑，不会被定时任务吞掉。

主动策略会从首次心跳开始累计从未聊天用户的空闲时间，并结合关系档案中的主动亲和度调整空闲和冷却：用户最近拒绝主动话题时更久保持安静，明确欢迎主动靠近时才会小幅提前。下一次心跳会参考上一次的关系重点和选择理由；最近一次结果可在“三级记忆管理”顶部查看。

在“桌宠设置 → 桌面感知与关心”可分别开启两种能力：

- “允许 Agent / 心跳按需理解一次屏幕”需要上方独立识图配置完整。普通聊天只有在当前问题确实需要桌面情境并由模型调用 `desktop_observe` 时读取一次；手动心跳可明确观察，定时心跳只有已经满足主动策略时才会请求图片。主进程选择鼠标所在显示器，生成不超过约 960px 宽的 JPEG 缩略图，以 `image_url` 只发送给识图端点；聊天 LLM 仅接收清洗后的受限文字。图片字节不会写入记忆、关系档案、工具文本、心跳日志或磁盘。
- “本机检测可见应用与新启动活动”仅在 Windows 使用固定的本机只读查询。服务先运行快速 `tasklist.exe /fo csv /nh`，再只对已知应用补充可见性详情；快速查询或任一详情查询失败时，统一改用固定 PowerShell `Get-Process` + `MainWindowHandle` 脚本校正可见窗口，绝不把只有后台进程的精简行当成正在使用。两条路径都只向后续逻辑提供白名单应用的粗粒度类别，窗口标题、PID 和原始输出不会进入模型、档案或日志。新类别或间隔 4 小时后的独立观察才会增加习惯证据，至少三次后才可能进入关系上下文。

一次性屏幕和应用信号都被当作低置信情境，而不是命令或用户事实。桌宠必须用“好像、可能、如果你正在……”表达，并允许用户纠正；网页右键、剪贴板和文件选择器仍是用户主动触发的电脑共读路径，不属于主动聊天。

## Agent 工具调用

普通聊天不再只是“模型生成一段文字”。配置支持 function calling 的模型后，主进程会提供以下稳定工具：

| 工具 | 用途 | 关键约束 |
| --- | --- | --- |
| `memory_search` | 按当前/历史/比较时态召回 L2/L3 | 只返回相关摘要和本机记忆 ID |
| `memory_store` | 保存用户明确要求记住的内容 | 最终内容从用户原话提取，进入 L2 |
| `self_profile` | 读取证据驱动的人格阶段与倾向 | 只读，模型不能直接改写人格 |
| `relationship_profile` | 读取关系阶段、稳定理解和关心方式 | 只读，结果仍允许用户纠正 |
| `desktop_observe` | 一次性屏幕与粗粒度应用感知 | 两个开关独立、默认关闭；图片不落盘 |
| `computer_*` | 网页/剪贴板/文件/应用、当前标签页与 Office 单步写入 | 只创建待确认预览，主进程再次校验参数；浏览器/Office 每次确认 |
| `computer_work_plan` | 编排 2～4 个现有白名单步骤 | 全部步骤先校验；一次展示一步，任一步拒绝/取消/失败即停止后续 |
| `pet_action` | 为本次回复选择一个身体动作 | 服从录音、拖拽、落地和强动作冷却 |

主进程共提供十五项稳定工具。一次 Agent 回合最多进行四轮模型—工具往返和十次实际工具调用。每个结果用标准 `role: tool + tool_call_id` 回填；`desktop_observe` 的图片只进入独立识图请求，聊天工具结果只包含清洗后的低置信文字和通道状态。网页共读正文不会获得这些工具；一轮回复最多创建一个单步预览或一个 2～4 步计划，用户授权后的真实执行结果会写回 L1，让下一轮知道刚才成功、取消还是被拒绝。顺序计划不执行开放式循环，也不撤回已完成步骤。

## 记忆生命周期

| 层级 | 定位 | 保存位置 | 进入下一层的条件 |
| --- | --- | --- | --- |
| L1 | 当前会话的逐条消息 | 进程内存 | 超过数量/时间阈值，或手动心跳 |
| L2 | 尚未抽象的对话事件 | 本地 JSON | 达到整理阈值，或手动心跳 |
| L3 | 事实、偏好、事件、反思 | 本地 JSON | 长期保留，相似记忆合并更新 |

从宠物右键菜单进入“三级记忆管理”，展开任意卡片即可查看完整内容和上游来源标识。L1 是当前会话上下文，只读且退出后丢失；L2/L3 的修正会保留原 ID、层级、创建时间、访问统计和来源链，只更新内容、类型、重要度、摘要与更新时间。删除只移除选中的持久记忆，不会跨层级联删除。

在聊天中明确说“请记住……”或“别忘了……”时，清理后的内容会立即持久化到 L2，而不是只在 L1 口头答应；之后仍由心跳整理为合适的 L3 事实、偏好或事件。

搜索和聊天上下文不会再用重要度或新鲜度补齐完全无关的记录：持久记忆至少要达到加权文本相关度 `0.75` 才参与排序和访问强化。记忆类型带有本地化检索词，因此心跳思考仍能用“近期偏好、重要的事、待跟进话题”这类通用关注点找到候选；确实没有匹配时搜索返回空列表，心跳会结合关系和短时情境决定安静，而不是为了完成额度强行找一条旧事。

控制面板搜索始终保留完整相关历史，方便审计和手工修正；聊天上下文则对事实与偏好使用查询时态门控。默认或询问“现在 / 目前 / 最新”时排除带“以前 / 过去 / 原计划”等明确历史线索的记录；询问历史时只取明确历史证据；询问“变化 / 前后 / 对比”时同时保留新旧证据。门控发生在结果截断和访问强化之前，因此被排除的旧值不会进入模型提示或增加访问次数。该规则仍是轻量显式词法判断，没有时态词的隐式冲突、同义表达和代词指代仍需后续版本链与冲突审核能力处理。

应用数据通过设置里的“打开数据目录”查看。记忆和最近心跳审计在 `data/memory-store.json`，人格状态在 `data/personality-profile.json`，关系理解在 `data/relationship-profile.json`，三组端点的公开设置在 `data/settings.json`，三份加密凭据在 `data/secrets.json`，用户导入模型在 `data/models/`。屏幕图片不属于本地数据文件，识图请求完成后只等待运行时回收。

陪伴式对话的行为原则、消息组织和研究依据见 [`docs/COMPANION_DIALOGUE.md`](./docs/COMPANION_DIALOGUE.md)。

## 语音说明

- 语音识别默认使用项目内的 sherpa-onnx 1.13.4 和 14M 中文流式 Zipformer。Renderer 采集单声道麦克风 PCM、做简单语音活动检测并下采样到 16 kHz；主进程只接受来源为宠物窗口、0.25～30 秒的 PCM16，再交给独立 worker 本机识别。录音不会发送到聊天模型或 TTS 服务。
- 说完保持约 1.2 秒安静会自动结束录音，也可再次点击麦克风手动结束；8 秒内没有检测到语音会给出明确提示。权限等待不计入录音时限，启动、静默和最长录音都有独立墙钟看门狗，麦克风轨道中断也会立即收尾；30 秒音频按采样数精确裁剪到主进程边界。
- 应用启动后会在后台非阻塞预热模型；文件校验状态与 worker 运行状态分开呈现。预热失败会显示原因但保留麦克风重试入口，下一次识别会重新初始化。进入“正在本地识别…”后再次点击会立即取消并忽略迟到结果；主进程初始化/交互各有 30 秒上限，Renderer 另有 35 秒看门狗。
- Electron 43 的 Chromium 设备端 `processLocally` 在当前环境缺少底层 Mojo 服务，不能作为可靠本地实现，因此本地模式不再调用它。“Chromium 兼容模式”仅作为可选后备，可能联网且稳定性取决于 Chromium 后端。
- 自动朗读默认使用操作系统已安装的本机语音，不需要 TTS Key，也不会发送回复文本。语速和识别语言同时用于选择合适的本机声音。
- 选择云端 TTS 后，主进程才会使用独立端点与凭据调用 `/audio/speech`。单次文本最多 2000 字，响应限制为 12MB MP3，超时为 60 秒；失败时自动回退本机语音。
- 新回复会停止正在播放的旧回复，晚到的旧云端结果不会覆盖当前语音。隐藏桌宠会停止输入与朗读，隐藏后返回的聊天结果不会在后台重新发声；重新唤醒或主动聊天显示桌宠后恢复正常。麦克风、本地模型或朗读方式异常时，文字聊天和三级记忆仍可使用。

## Live2D 模型与动作

[`Live2DPetAdapter`](./src/renderer/live2d-pet-adapter.ts) 使用 PixiJS 8、Cubism Framework 和官方 Cubism Core 5.1 渲染 `.moc3`，并负责自动取景、眨眼、物理、视线跟随、口型和 motion。`DesktopMovementController` 只负责 Electron 窗口在桌面的真实位移，因此模型切换不会影响三级记忆、心跳、拖拽和自主漫游。

项目内置 9 套经过资源边界检查的模型：

| 模型 | 来源 | motion / 表情 | 初始气质 |
| --- | --- | ---: | --- |

| Mao | Live2D 官方样例 | 8 / 8 | 活泼爱玩 |
| Wanko | Live2D 官方宠物样例 | 12 / 0 | 热情直率 |
| Haru | Live2D 官方样例 | 6 / 8 | 开朗细腻 |
| Mark | Live2D 官方卡通样例 | 6 / 0 | 安静稳重 |
| Nana | CC BY-SA 4.0 第三方模型 | 0 / 0 | 清醒温和 |
| Rice | Live2D 官方样例 | 4 / 0 | 认真慢热 |
| Cyannyan | CC BY-SA 4.0 第三方模型 | 0 / 16 | 俏皮机敏 |
| 小云 | CC BY-NC-SA 4.0 第三方模型 | 0 / 18 | 软萌主动 |

导入自己的模型：

1. 准备一个 Cubism 3/4/5 模型文件夹，其中必须且只能有一个 `.model3.json`，并包含它引用的 `.moc3` 和全部贴图。
2. 右键宠物选择“模型与动作 → 导入 Live2D 模型…”，或打开“桌宠设置 → Live2D 模型与动作”。
3. 选择整个模型文件夹。校验成功后，所需资源会复制到 `data/models/imported/` 并立即切换；原目录之后可以移动。

适配器把 `Idle` 作为循环待机。真实 motion 只在动作语义经过校验时绑定，缺少匹配资源时按语义播放平滑起止的程序化全身变形，并叠加模型实际存在的眼睛、头身与耳朵参数；一次性资源动作从实际 `.motion3.json` 的 `Meta.Duration` 读取 600～12,000ms 播放时长。情绪 expression、标准/旧式口型兜底、全局视线和无嘴部参数时的发言身体脉冲都在运行时按真实参数能力选择。自动反应会避让录音、拖拽、下落和落地，强动作有 12 秒冷却；用户手动预览会清空待执行自动动作，并从最近一次手动动作起固定抑制自动动作 12 秒。

详细目录约定、限制、动作组织和授权要求见 [`docs/LIVE2D_MODEL_GUIDE.md`](./docs/LIVE2D_MODEL_GUIDE.md)。本地 UI 和 Electron 验收图生成在 `output/`，该目录包含测试运行数据且不纳入版本控制。

## 隐私与安全边界

- 默认所有记忆、人格证据和设置只写入本机应用数据目录。
- 启用聊天模型时，当前输入、经过相关性与时态门控的少量记忆、已达到门槛的人格状态，以及用户通过网页右键、剪贴板或原生文件选择器明确共享的内容会发送到聊天端点。屏幕图片只发送到独立识图端点，聊天 LLM 只获得受限视觉摘要；只有主动选择云端 TTS 时，助手回复文本才会发送到 TTS 端点。聊天、识图与 TTS 的 Base URL、模型和 API Key 互不共用。
- 渲染进程启用了上下文隔离、沙箱、CSP，并通过白名单 IPC 与主进程通信。
- 记忆、人格复盘对话和电脑上下文在模型提示中被明确标记为数据而不是指令；网页标签闭合字符会转义，页面文字不能直接触发电脑工具。
- 浏览器桥接仅绑定本机回环地址并校验随机令牌和扩展来源。工具调用只接受主进程保存的操作 ID，Renderer 不回传 URL、文本、文件路径或应用名，避免确认后参数被替换。

更细的模块职责和心跳流程见 [`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md)。
